## What this repo is

Fram3 is a Flutter-inspired declarative UI framework for Unity built on UIToolkit. Elements are immutable descriptions of UI; the renderer diffs and patches the UIToolkit visual tree each frame. The Unity package lives at `Packages/com.fram3.ui/`. The `.csproj` files at the repo root are generated by Unity and should not be edited.

## Commands

All test commands run against the standalone .NET test project, not the Unity Test Runner.

```bash
# Working directory for all test commands:
cd Packages/com.fram3.ui/Tests

dotnet restore Fram3.UI.Tests.csproj
dotnet build Fram3.UI.Tests.csproj --configuration Release
dotnet test Fram3.UI.Tests.csproj --configuration Release --verbosity normal

# Run a single test class:
dotnet test Fram3.UI.Tests.csproj --filter "FullyQualifiedName~TreeDifferTests"
```

There is no separate lint or format step — the project uses standard C# nullable annotations (`#nullable enable`) throughout.

## Architecture

### Package layout

```
Packages/com.fram3.ui/
  Runtime/        # All library source (compiled by Unity and by the test project)
  Tests/          # NUnit tests + mocks (dotnet net9.0, no Unity dependency)
  Editor/         # Unity Editor-only assembly
tools/
  DocBuild/       # Builds the Runtime with XML doc output for doc generation
  DocExtractor/   # Converts XML docs to MDX and pushes to the docs repo
```

### Runtime subsystems

| Directory | Responsibility |
|---|---|
| `Runtime/Core/` | Public element base types: `Element`, `StatelessElement`, `StatefulElement`, `InheritedElement`, `State`, `BuildContext`, `Key`, `ValueNotifier` |
| `Runtime/Core/Internal/` | Reconciliation engine: `Node`, `NodeExpander` (mount/unmount/expand), `TreeDiffer`, `TreePatcher`, `RebuildScheduler`, `DiffOp` |
| `Runtime/Rendering/` | Public entry points: `Renderer` (one per UIDocument), `AppRoot` (MonoBehaviour helper), `RootElement` |
| `Runtime/Rendering/Internal/` | `ElementPainter` (creates/updates native `VisualElement`s) — split into partial classes by concern; `RenderHandle` |
| `Runtime/Elements/` | All built-in elements: `Layout/`, `Content/`, `Input/`, `Gesture/`, `Animation/`, `Theme/`, `State/` |
| `Runtime/Animation/` | `AnimationController`, `AnimationSystem`, `AnimatedValue`, `Curve`, `Lerp` |
| `Runtime/GlobalState/` | `Store` (Redux), `Cubit` (BLoC), `PersistentStore`, `PlayerPrefsAdapter` |
| `Runtime/Navigation/` | `Navigator` (in-tree), `SceneNavigator` (Unity scene loading) |
| `Runtime/Styling/` | `FrameColor`, `EdgeInsets`, `BorderRadius`, `BoxDecoration`, `TextStyle`, `Alignment` |

### How the reconciler works

`Renderer.Mount` builds an initial `Node` tree from the element tree, then `Renderer.Tick` flushes rebuild requests queued by `RebuildScheduler`. For each dirty node, `NodeExpander` re-expands it, `TreeDiffer` computes a list of `DiffOp`s (add/remove/update), and `TreePatcher` applies them. `ElementPainter` translates leaf-level element changes into mutations on real `UnityEngine.UIElements.VisualElement` instances.

### Unity isolation in tests

Tests compile without Unity by including `FRAM3_PURE_TESTS` as a `DefineConstants`. `Tests/Mocks/UnityTypeStubs.cs` provides stub implementations of Unity/UIToolkit types guarded by `#if FRAM3_PURE_TESTS || FRAM3_DOC_BUILD`. Runtime code that creates native `VisualElement`s (primarily `ElementPainter` and `SpinnerElement`) also uses `#if !FRAM3_PURE_TESTS` guards. Never remove these guards.

### API documentation for framework consumers

`Packages/com.fram3.ui/AGENT.md` contains the complete public API reference (element constructors, styling types, state management, navigation, animation). Keep it updated whenever public constructor signatures, property names, or behavioral contracts change. The sync-docs CI workflow generates MDX from XML doc comments and pushes it to the separate `fram3-docs` repo.

## CI

- **ci.yml** — runs `dotnet test` on push/PR to `main`
- **release.yml** — creates a GitHub release from the version in `package.json` on push to `main`
- **sync-docs.yml** — builds XML docs via `tools/DocBuild`, runs `tools/DocExtractor`, opens a PR in the `fram3-docs` repo
